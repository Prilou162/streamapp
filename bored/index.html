Contexto

Problema y oportunidad en streamers mid-tier.

El ecosistema de creadores está lleno de herramientas pensadas para grandes streamers o campañas masivas. Pero la franja de 80–300 viewers concurrentes en LATAM queda subatendida, tanto para las marcas como para los creadores.

Dolor del lado del streamer
Streamers medios tienen comunidad real, pero les cuesta traducir eso en acuerdos claros.
  • Dificultad para presentarse profesionalmente ante marcas.
  • Falta de claridad sobre qué ofrecer y cómo valorarlo.
  • Sin un lugar central para mostrar métricas, campañas y redes.
Dolor del lado de la marca
Marcas con presupuesto moderado buscan alcance nicho, pero los procesos son artesanales.
  • Les cuesta identificar streamers alineados a su rango y categoría.
  • Falta de fichas estándar para comparar creadores.
  • Poca estructura para testear campañas chicas a escala.
Solución propuesta

Un MVP que ya muestra el flujo completo de valor.

STREAMLATAM no es un mockup. Es un recorrido real donde un streamer puede ver su snapshot, recibir campañas sugeridas, aplicar, y mostrar un perfil público orientado a sponsors.

Pantalla
Dashboard del streamer

Snapshot de métricas, insights para sponsors y progreso hacia objetivos mensuales (postulaciones y campañas aprobadas).

Pantalla
Listado de campañas

Campañas filtrables por región, tipo de pago, tipo de activación y categoría, con un preview lateral que se actualiza al hacer clic.

Pantalla
Detalle de campaña

Vista corporate-friendly donde se explican requisitos, flujo y condiciones de la campaña desde la perspectiva del streamer y de la marca.

Pantalla
Mis postulaciones

Listado simple de aplicaciones demo que permite explicar al usuario cómo ver el estado de cada campaña.

Pantalla
Perfil del streamer

Ficha editable demo con datos base del canal, redes sociales, historial de campañas y horarios habituales de stream.

Pantalla
Perfil público (lado marca)

Vista pública que simula cómo una marca vería al streamer antes de aprobar una activación, usando los mismos datos que el dashboard.

Qué se vende

Paquete completo: idea, MVP, código y narrativa.

La intención no es operar el SaaS, sino entregar un paquete bien armado para que otro equipo lo lleve a producción, lo integre a su portfolio o lo use como base para un nuevo producto en creator economy.

Incluye, a nivel producto / diseño
  • Concepto completo de plataforma para streamers mid-tier de LATAM.
  • Arquitectura básica de navegación (sidebar + topbar + vistas principales).
  • 6 pantallas diseñadas y alineadas visualmente.
  • Copy en español orientado a streamers y marcas.
  • Demostración clara de flujos clave: descubrir campaña, entenderla, aplicar, mostrar perfil.
Incluye, a nivel técnico
  • Código fuente HTML + CSS + JS (sin dependencias complejas).
  • Estructura lista para migrar a framework (React / Next / Vue / etc.).
  • Uso de localStorage para simular datos del streamer.
  • Componentes reutilizables de layout (tarjetas, pills, listas, grids).
También se puede incluir (a acordar):
  • Documento corto con roadmap sugerido a SaaS (MVP técnico real).
  • Ideas de pricing, take rate y posibles modelos de comisión.
  • Listado de próximas features lógicas (ej: login, panel para marcas, onboarding, etc.).
  • Pitch deck en formato editable (según acuerdo).
Fit potencial

Quién puede aprovechar mejor esta adquisición.

El MVP está pensado para ser un atajo: ahorrar meses de validación, diseño y armado de narrativa en un espacio donde los creadores ya existen, pero el tooling sigue fragmentado.

Que quieran sumar una vertical de creator economy sin empezar desde cero.

  • Pueden integrarlo a su portfolio de productos propios.
  • Les sirve como demo para conseguir marcas o fondos.
  • Les ahorra tiempo de discovery + UX inicial.
Startups / empresas en marketing de influencers

Que ya trabajan con creadores y quieran explorar una solución específica para streamers mid-tier.

  • Fácil de adaptar a su base de datos actual.
  • Pueden rebrandear y llevarlo a producción con su equipo.
  • Permite tests rápidos en un nuevo segmento sin rehacer UX.
